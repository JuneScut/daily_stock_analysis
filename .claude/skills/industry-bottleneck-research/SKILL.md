# Industry Bottleneck Research

使用“紫苏葉 / Serenity 瓶颈理论”做行业深挖：当大趋势爆发时，优先寻找产业链里真正限制放量、最难替代、价值占比不一定最高但弹性可能最大的“卡脖子小环节”，再递归追踪相关企业和上游约束。

Reference: https://www.tmtpost.com/8020233.html

## Reuse Project Capabilities

优先复用本项目已有能力做证据采集；缺口再用外部搜索或标注“待核实”。

在本仓库内调用项目工具时，优先使用 `.venv/bin/python`，确保加载仓库虚拟环境中的 `fake-useragent`、`efinance`、`akshare`、`baostock` 等数据源依赖。若直接使用系统 `python` 出现 `ModuleNotFoundError` 或数据源导入失败，先切回 `.venv/bin/python` 验证，不要把它误判成工具能力缺失。

| 个股最新新闻、订单、客户、政策、风险事件 | Agent 工具 `search_stock_news` |

## Memory Loop

本 skill 必须随研究进化。每次做行业瓶颈研究时：

1. 先读取 `memory/INDEX.md`。
2. 若存在对应行业文件，读取 `memory/<industry-slug>.md`，复用已有判断，并标注哪些判断被新证据强化、削弱或推翻。
3. 研究结束后更新或创建对应行业文件；不要只把洞察留在对话里。
4. 更新 `memory/INDEX.md` 的行业条目、最后研究日期、核心瓶颈和最高优先级待验证问题。

行业文件只保存可复用判断，不保存长篇新闻摘录。把结论分成：

- `Durable Insights`：跨周期仍有用的产业链结构、认证门槛、扩产约束。
- `Current Hypotheses`：依赖当期订单、价格、政策、客户节奏的判断。
- `Company Map`：公司到瓶颈环节的映射及置信度。
- `Disconfirmed / Weakened`：已被证伪或明显削弱的旧判断。
- `Open Questions`：下次研究优先核验的问题。

## When To Use

- 用户指定行业、赛道、主题或产业链，要求“深挖”“卡脖子”“瓶颈”“相关企业”“隐藏标的”“产业链受益方”。
- 用户问某个热点行业除了龙头之外还有哪些上游环节或小市值弹性公司。
- 用户要求比较行业链条里哪个环节最稀缺、最难国产替代、最可能受益于放量。

不要在用户只想要单只股票常规技术面分析时使用；这时最多把行业结论作为辅助上下文。

## Core Principle

不要从最显眼的终端公司开始下结论。先问：“如果这个行业需求放大 3-10 倍，最先不够用、最难扩产、最难认证或最难替代的环节是什么？”

把 KOL 战绩、社媒情绪和短期涨幅当作待验证信号，不当作证据。最终结论必须落到产业链位置、技术/工艺壁垒、客户验证、产能弹性、收入暴露和替代风险。

## Workflow

1. **读取历史洞察**
   - 读取 `memory/INDEX.md` 和对应行业文件。
   - 如果没有历史文件，先创建研究假设，不要假装已有积累。

2. **界定行业边界**
   - 明确行业/主题、区域市场、时间窗口、下游需求驱动。
   - 若用户没有指定市场，默认同时考虑 A/H/美股与关键海外未上市企业，并标注上市地。

3. **拆产业链**
   - 按“原材料 -> 设备/工具 -> 核心零部件 -> 模组/系统 -> 整机/应用 -> 渠道/运营”拆层。
   - 对每层写出关键产品、成本占比、技术指标、扩产周期、认证门槛和主要客户。

4. **找卡脖子环节**
   对每个环节问：
   - 放量时是否会先短缺？
   - 是否依赖少数供应商、特殊材料、核心设备、专利、良率或认证？
   - 国产替代/全球替代难度如何？
   - 下游客户切换供应商的成本高不高？
   - 单价、毛利率、BOM 占比和利润弹性是否足够？
   - 需求被证伪时，产能和估值是否会反噬？

5. **递归深挖**
   - 对每个候选瓶颈继续追问“这个瓶颈的瓶颈是什么”。
   - 至少深挖 2 层：行业瓶颈 -> 企业产品 -> 企业自己的上游/设备/材料/客户验证瓶颈。
   - 如果只能找到题材关联而找不到真实约束，降低结论置信度。

6. **建立企业清单**
   对每家公司记录：
   - 公司名称、代码、上市地/是否未上市
   - 对应瓶颈环节和具体产品
   - 收入或利润暴露度；若未知，明确写“待核实”
   - 核心客户、竞争对手、替代品
   - 产能、良率、认证、专利、成本或交付优势
   - 催化剂、验证指标、主要风险

7. **用项目能力验证候选公司**
   - 对候选上市公司优先调用/参考 `get_stock_info`、`search_comprehensive_intel`、`search_stock_news`。
   - A 股行业热度用 `get_sector_rankings` 辅助判断，不把涨幅榜当产业瓶颈证据。
   - 行情、估值和资金只用于“交易拥挤/验证节奏”判断，不替代产业链判断。

8. **排序与分层**
   - 第一层：真实瓶颈、客户验证明确、收入暴露高。
   - 第二层：可能受益，但瓶颈程度或收入暴露待验证。
   - 第三层：题材映射弱、仅情绪扩散或间接受益。

9. **输出可验证结论**
   - 区分“产业逻辑成立”“企业映射成立”“投资性价比成立”三件事。
   - 不给无依据的买入建议；必须写清需要继续验证的数据和反例。

10. **沉淀行业洞察**
   - 更新 `memory/<industry-slug>.md` 和 `memory/INDEX.md`。
   - 明确本次新增、强化、削弱、推翻的判断。
   - 若没有写入 memory，最终答复必须说明原因。

## Quality Bar

- 至少覆盖 5 个产业链环节，除非行业本身很窄。
- 至少给出 3 类反证：需求、供给、技术路线。
- 所有公司映射都要说明“为什么是这个环节”，不能只列股票名。
- 涉及实时行情、估值、市占率、订单、客户或政策时，必须用最新可查证信息，并标注来源时间。
- 结尾保留“非投资建议；以上用于产业链研究和风险识别”。
